# ADR-001: Вибір VictoriaMetrics

- Статус: Accepted
- Дата: 2026-03-07

## Контекст
Потрібна проста і надійна система збору та зберігання метрик для KDI з мінімальними операційними витратами.

## Рішення
Використовуємо `VictoriaMetrics` як центральне TSDB-сховище метрик.

## Обґрунтування
- Простий single-binary запуск
- Низькі вимоги до ресурсів у single-node режимі
- Сумісність із Prometheus scrape-моделлю
- Добра інтеграція з Grafana

## Наслідки
- На старті працюємо в single-node топології
- Питання scale-out вирішуємо окремо ADR при зростанні навантаження
